# Release & Pre-release

## Versioning
- Mod version: `<mod_version>` (e.g. `1.0.0`)
- Tag format: `v<mod_version>+mc<mc_version>` (e.g. `v1.0.0+mc1.21.3`)
- Jar name: `cohesion-<mod_version>-<mc_version>.jar` (generated by build.gradle)

## Release naming
- **Title**: `Cohesion <mod_version> for MC <mc_version>`
- **Description**: One sentence, e.g. `Cohesion port to Minecraft <mc_version>.`

## Steps

1. Build the mod: `./gradlew build`
2. Push the branch: `git push origin <local_branch>:mc/<mc_version>`
3. Create and push the tag: `git tag v<mod_version>+mc<mc_version> && git push origin v<mod_version>+mc<mc_version>`
4. Create the release:
```bash
gh release create v<mod_version>+mc<mc_version> \
  build/libs/cohesion-<mod_version>-<mc_version>.jar \
  --target mc/<mc_version> \
  --title "Cohesion <mod_version> for MC <mc_version>" \
  --prerelease \
  --notes "Cohesion port to Minecraft <mc_version>."
```

## Pre-release vs stable
- Always create as `--prerelease` first
- Promote to stable once validated: `gh release edit v<mod_version>+mc<mc_version> --prerelease=false`

## Rules
- One release per MC version branch — branches are never merged
- Attach only the remapped jar (not `-sources.jar`)
- Do not include dependency lists, install instructions, or changelogs in the description — keep it to one sentence
